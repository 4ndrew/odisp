ODISP
Что такое ODISP?
 

ODISP (Object Dispatcher) это полностью асинхронная транспортная система без ведения сессий. За основу были взяты некоторые идеи реализации микроядерных операционных систем и систем организации распределенных вычислений.

В основе модели диспетчера объектов лежит метод передачи сообщений между отдельными, логически цельными, модулями. Каждый модуль имеет связь с диспетчером и почтовый ящик для приходящих сообщений. Сообщения между модулями служат исключительно для передачи данных (уведомления).

Какова цель?
 

Цель - создать гибкую расширяемую систему ориентированную на реакцию на внешние воздействия.

Применение?
 На данный момент ODISP используется в качестве ядра для моего рабочего проекта в НПП "Новел-ИЛ". На его основе реализуется клиент-серверная система мониторинга и управления оборудованием компании.
Каково состояние проекта?
 Проект находится в активной разработке начиная с октября 2003 года. На данный момент это полностью функционирующая система реализующая поставленные перед ней задачи. В ближайшем будущем планируется написание стыка ODISP-CORBA.
Кто автор?

Лицензия?
 ODISP распространяется на основе лицензии BSD.
Документация?
 Основное описание системы и (немного) вольное описание логики работы отдельных компонент.
Где взять?
 Рабочая версия.
Зависимости?
 ODISP зависит от анализатора XML от XLang, который включён в архив. Так же его можно собрать самостоятельно взяв исходный код со страницы XLang (версия W3).
